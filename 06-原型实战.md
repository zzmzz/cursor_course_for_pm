# 第 6 课：原型实战 — 搭建完整的产品原型

## 本课目标

综合运用前 5 课的所有技能，从零搭建一个完整的「待办事项」应用。

---

## 为什么选「待办事项」？

待办事项（Todo App）是一个麻雀虽小五脏俱全的项目，它包含了大部分产品原型需要的元素：

- 输入框（添加任务）
- 列表展示（展示所有任务）
- 状态切换（标记完成 / 未完成）
- 删除操作
- 数据持久化（刷新页面后数据还在）

搞定这个，其他更复杂的原型也能照葫芦画瓢。

---

## Step 1：用 Agent 生成基础框架

### 操作步骤

1. 在桌面创建一个新文件夹 `todo-app`
2. 用 Cursor 打开这个文件夹（File → Open Folder）
3. 按 `Cmd+I` 打开 Composer，选择 **Agent** 模式
4. 输入以下需求描述：

> 帮我创建一个「待办事项」应用。要求：
>
> 1. 页面顶部有标题"我的待办事项"
> 2. 标题下方有一个输入框和「添加」按钮，用来添加新任务
> 3. 下方展示任务列表，每个任务包含：
>    - 左侧一个复选框，勾选表示完成
>    - 任务文字（完成后加删除线）
>    - 右侧一个删除按钮
> 4. 底部显示统计：共 X 项，已完成 X 项
> 5. 数据保存在浏览器本地存储中（刷新不丢失）
> 6. 视觉风格：简洁现代，居中卡片布局，圆角，浅色背景
> 7. 请创建一个完整的项目，可以用 npm run dev 启动

5. 按回车，等待 AI 完成
6. AI 可能会创建多个文件，逐步 Accept 每一步操作

### 验证结果

1. 打开终端（`` Ctrl+` ``）
2. 输入 `npm install`，等待完成
3. 输入 `npm run dev`
4. 在浏览器中打开终端显示的地址
5. 你应该能看到一个能正常工作的待办事项页面

**检查清单：**
- [ ] 能输入文字并添加任务？
- [ ] 能勾选完成任务？（文字有删除线）
- [ ] 能删除任务？
- [ ] 底部统计数字正确？
- [ ] 刷新页面后任务还在？

## Step 2：用 Composer 迭代添加功能

基础版跑起来了，现在我们加点功能。继续在 Composer 里对话：

### 添加功能 1：任务分类

> 给每个任务加上分类标签功能。添加任务时可以选择分类：工作、生活、学习。在列表上方加上分类筛选按钮，点击后只显示对应分类的任务。

### 添加功能 2：编辑任务

> 双击任务文字可以进入编辑模式，修改任务内容后按回车保存。

### 添加功能 3：拖拽排序

> 让任务列表支持拖拽排序，用户可以通过拖动来调整任务顺序。

每次添加完一个功能：
1. 在浏览器里刷新查看效果
2. 测试功能是否正常
3. **满意的话，去 Source Control 面板提交一次 Git**（写清楚这次改了什么，比如"添加任务分类功能"）

> 重要：每次满意的修改都要提交 Git，这是你的「存档点」。后面出问题可以回到这里。

---

## Step 3：用 Composer 美化界面

现在功能差不多了，我们来美化界面。继续在 Composer 里对话就行。

### 美化 1：修改颜色主题

> 把整个配色方案改成莫兰迪色系（低饱和度的高级感配色）

### 美化 2：优化卡片样式

> 给卡片加上轻微的阴影效果和悬停动画，让卡片有浮起感

### 美化 3：参考某个设计风格

如果你有喜欢的设计参考，直接描述：

> 我希望这个应用的视觉风格参考 Apple 的设计语言：大量留白，SF 字体风格，极简的图标，轻量的阴影。帮我调整整体样式。

## Step 4：处理报错

在搭建过程中，你很可能会遇到报错。**这完全正常**，就连专业的研发也天天和报错打交道。重要的是知道怎么应对。

### 情况 1：终端里出现红色报错文字

1. 看报错信息（通常最后几行会说明问题所在）
2. 全部选中报错文字，复制
3. 在 Chat（`Cmd+L`）里粘贴，然后说：「这个报错是什么意思？帮我修复」
4. AI 会分析报错原因并给出修复方案
5. 点 Apply 应用修复

### 情况 2：页面显示空白或样式错乱

1. 在 Chat 里说：「页面显示不正常，[描述你看到的问题]，帮我检查和修复」
2. 用 `@codebase` 让 AI 检查整个项目

### 情况 3：AI 改了半天越改越坏

这是最常见的「翻车」场景。解决办法：

1. 打开 Source Control 面板
2. 找到出问题的文件
3. 右键 → **Discard Changes**
4. 文件恢复到上次提交的状态
5. 换一种方式向 AI 描述你的需求，重新来过

> 如果你想回退整个项目到之前某次提交，可以在终端里输入 `git log --oneline` 查看提交历史，然后告诉 AI 你想回退到哪个版本。

## 完整的工作流小结

通过这个实战，你掌握了一个完整的原型搭建工作流：

```
① Agent 生成基础框架
        ↓
② 浏览器预览验证
        ↓
③ Composer 对话迭代功能和美化
        ↓
④ 每次满意后 Git 提交
        ↓
⑤ 遇到问题 → 贴报错给 AI / Git 回退
        ↓
⑥ 循环 ②~⑤ 直到满意
```

这个流程适用于任何产品原型的搭建——不管是表单页面、后台管理系统，还是小程序的前端界面。

1. 用 Agent 生成基础框架
2. 至少做 5 次 Composer 迭代（功能 + 美化）
3. 至少提交 3 次 Git
4. 在浏览器里截图，记录你的成果
