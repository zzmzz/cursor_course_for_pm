# 第 5 课：理解项目与终端基础

## 本课目标

能看懂一个前端项目的目录结构，会用终端启动项目在浏览器预览，会用 Git 管理修改。

---

## 为什么要学这些？

前 4 课你已经能让 AI 帮你生成页面和文档了。但如果 AI 生成了一个稍微复杂一点的项目（不是一个单独的 HTML 文件，而是一个有多个文件的正式项目），你需要知道：

1. 这些文件是怎么组织的（项目结构）
2. 怎么把它跑起来看效果（终端命令）
3. 怎么安全地管理你的修改（Git）

别紧张，这些比你想象的简单。

---

## 常见前端项目目录结构

当 AI 帮你用 React、Vue 或 Next.js 等框架（框架就是「别人搭好的半成品，你往里面填内容就行」）生成项目时，你通常会看到这样的目录结构：

```
my-project/
├── src/                    ← 源代码，最核心的文件夹
│   ├── pages/              ← 页面文件，一个文件对应一个页面
│   ├── components/         ← 组件，可复用的页面小零件（按钮、卡片、弹窗等）
│   ├── App.tsx             ← 应用的「总入口」
│   └── index.tsx           ← 启动文件
├── public/                 ← 静态资源（图片、图标等）
├── package.json            ← 项目配置文件（记录了项目名、依赖等信息）
├── node_modules/           ← 第三方库（自动下载的，不用管它）
└── README.md               ← 项目说明
```

### 关键文件 / 文件夹解释

| 名称 | 干什么的 | 类比 |
|------|---------|------|
| `src/` | 放你实际写的代码 | Figma 项目里的画板 |
| `pages/` | 每个文件就是一个页面 | 每个画板就是一个页面 |
| `components/` | 可以在多个页面中复用的小模块 | Figma 的组件库 |
| `package.json` | 记录项目信息和所需的依赖库 | App 的「关于」页面 |
| `node_modules/` | 自动下载的第三方代码库（很大，别动它） | 系统自带的底层框架 |
| `public/` | 放图片、图标等不需要处理的文件 | 设计稿里的切图素材 |

> 记不住没关系，你随时可以问 AI：`@codebase 这个项目的文件结构是怎样的？每个文件夹都放什么？`

## 快速导航技巧

### Cmd+P：按文件名搜索

按 `Cmd+P`（Mac）或 `Ctrl+P`（Windows），弹出一个搜索框，直接输入文件名的一部分，就能快速跳转到那个文件。

比如你想找登录页面，输入 `login`，相关的文件就会出现在列表里。

> 这比在文件树里一个个点开文件夹找快多了。

### Cmd+Shift+F：全局搜关键词

按 `Cmd+Shift+F`（Mac）或 `Ctrl+Shift+F`（Windows），打开全局搜索面板。

这个功能特别实用的场景：**你看到页面上有一段文字"请输入密码"，想找到对应的代码在哪里** → 在全局搜索里搜 `请输入密码`，Cursor 就会告诉你这段文字在哪个文件的哪一行。

## 终端基础

### 终端是什么？

终端（Terminal）就是一个**文字版的操作界面**——你输入一行文字指令，电脑就会执行对应的操作。

> 类比理解：你平时在电脑上双击图标来打开 App，那是图形化操作。终端就是用打字的方式来完成同样的事情。

### 打开终端

在 Cursor 里，按 `` Ctrl+` ``（反引号，就是键盘左上角 Esc 键下面那个键），底部就会弹出一个终端面板。

### 你只需要记住 3 个命令

| 命令 | 做什么 | 类比 |
|------|--------|------|
| `npm install` | 安装项目需要的依赖库 | 给新手机装必要的 App |
| `npm run dev` | 启动项目，让它跑起来 | 点击「运行」按钮 |
| `Ctrl+C` | 停止正在运行的项目 | 点击「停止」按钮 |

### 操作流程

1. 打开终端（`` Ctrl+` ``）
2. 输入 `npm install`，按回车，等待它跑完（会看到一堆下载信息，别管它）
3. 输入 `npm run dev`，按回车
4. 终端里会显示一个地址，通常是 `http://localhost:3000` 或 `http://localhost:5173`
5. 按住 `Cmd` 点击这个地址（或者手动复制到浏览器地址栏）
6. 浏览器里就能看到你的项目页面了

> `localhost` 的意思是「本地主机」，就是你自己的电脑。`3000` 是端口号，相当于门牌号。所以 `localhost:3000` 就是说「在你自己电脑的 3000 号门打开这个网页」。

### 想停止项目？

在终端里按 `Ctrl+C` 就行了。

## Git 图形化操作

### Git 是什么？

Git 是一个**代码版本管理工具**。

> 类比理解：你用 Figma 时能看到设计稿的版本历史，可以查看之前的版本，甚至回退到某个旧版本。Git 对代码做的就是同样的事情。

为什么你需要知道 Git？因为它能让你**不怕搞坏代码**——就算 AI 帮你改出了问题，你随时可以回退到上一个正常的版本。

### 在 Cursor 里操作 Git

好消息是：你不需要记 Git 的命令行用法。Cursor 提供了图形化界面来操作 Git。

#### 查看修改了哪些文件

1. 点击左侧边栏的「分支图标」（Source Control 面板，第三个图标）
2. 你会看到所有被修改过的文件列表
3. 点击某个文件，可以看到具体改了什么（左边是旧版本，右边是新版本）

#### 提交修改（保存一个版本）

「提交」（Commit）就是告诉 Git："我现在的修改没问题了，帮我保存一个版本。"

1. 在 Source Control 面板里，点击修改文件旁边的 `+` 号（暂存修改）
2. 在顶部的输入框里写一句说明，比如"修改了首页标题"
3. 点击 **Commit**（提交）按钮（✓ 图标）

#### 撤销修改（回到上一个版本）

如果你改坏了什么：

1. 在 Source Control 面板里，找到那个改坏的文件
2. 右键 → **Discard Changes**（丢弃修改）
3. 文件就会恢复到上次提交时的状态

> 重要：养成习惯——每次 AI 帮你做了一步你满意的修改，就提交一次。这样万一后面出问题，你可以精确地回退到任何一步。

## 课后练习

1. 用 Agent 模式（Cmd+I）让 AI 帮你创建一个 React 或 Vite 项目（提示词：「帮我用 Vite 创建一个简单的待办事项应用」）
2. 在终端里运行 `npm install` 和 `npm run dev`，在浏览器里打开查看
3. 用 Cmd+P 搜索文件，找到首页文件
4. 用 Cmd+Shift+F 搜索页面上的某段文字，定位到对应代码
5. 做一次提交：在 Source Control 面板里提交你的修改
6. 在 Chat 里让 AI 帮你改点什么，Apply 后试试 Discard Changes 撤销
