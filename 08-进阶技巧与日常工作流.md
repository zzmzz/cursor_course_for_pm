## Rules 配置 — 给 AI 设定「人设」

### 什么是 Rules？

Rules 就是你预先告诉 AI 的「行为规则」。设好之后，AI 在每次对话中都会自动遵守这些规则，不需要你每次重复说。

> 类比理解：就像你给实习生写了一份工作手册——"回复客户时用正式语气"、"方案里必须包含风险评估"——实习生每次工作都会参考这份手册。

### 全局 Rules

全局 Rules 对你所有项目都生效。设置方法：

1. 按 `Cmd+Shift+P`（打开命令面板）
2. 输入 `Cursor Settings`，选择 **Cursor Settings**
3. 找到 **Rules for AI** 或 **General Rules** 部分
4. 在里面输入你的规则

**推荐产品经理设置的全局 Rules：**

```
- 用中文回答所有问题
- 解释技术概念时，用产品经理能理解的语言，避免技术黑话
- 给出代码修改建议时，同时说明这个修改对用户体验的影响
- 写文档时使用 Markdown 格式
- 生成前端页面时，默认使用简洁现代的设计风格
```

### 项目级 Rules（.cursorrules 文件）

如果你只想让某些规则在特定项目里生效，可以在项目根目录下创建一个叫 `.cursorrules` 的文件。

比如你在搭建一个电商原型项目，可以在 `.cursorrules` 里写：

```
这是一个电商产品原型项目。
- 所有页面风格统一：使用红色作为主色调，白色背景，12px 圆角
- 中文文案，面向国内 25-35 岁女性用户
- 交互设计参考淘宝和小红书的风格
- 按钮都要有 hover 和 click 的反馈动画
```

AI 在这个项目中的所有对话都会自动参考这些规则。

## 与研发协作

学会 Cursor 之后，你和研发的协作会变得更顺畅。以下是几个实用场景：

### 场景 1：用 Cursor 阅读 PR（Pull Request）

PR 就是研发提交的代码修改申请，通常需要 Review（审查）后才能合并到主代码库。

作为 PM，有时你需要看看研发改了什么。在 Cursor 里：

1. 用 Chat 问：`@codebase` 最近的代码改动涉及了哪些功能？
2. 或者直接在 Source Control 面板查看改动列表，点击某个文件查看 diff

> 你不需要看懂每一行代码，但可以让 AI 帮你总结：「这次改动对用户有什么影响？」

### 场景 2：定位 Bug 相关的代码

用户反馈了一个 Bug："点击提交按钮后没有反应"。你可以：

1. 在 Cursor 里按 `Cmd+Shift+F`，搜索"提交"
2. 找到相关代码后，选中，按 `Cmd+L`，问 AI：「这段代码是处理提交按钮点击的逻辑，用户说点击后没有反应，可能是什么原因？」
3. 把 AI 的分析结果整理成 Bug 描述，发给研发

这比你只说"按钮点不了"有用得多——研发会觉得"这个 PM 太专业了"。

### 场景 3：用 AI 生成测试用例

> "帮我为'用户注册'功能生成一份测试用例表格，包含正常流程和异常流程。"

AI 能生成非常完整的测试用例，你只需要审查和补充业务细节。

## 常见问题 FAQ

### Q：Cursor 是免费的吗？

Cursor 有免费版和付费版：
- **免费版**（Hobby）：每月有一定次数的 AI 对话额度，基础功能都可以用
- **付费版**（Pro）：每月约 $20，AI 对话次数更多，可以使用更强的模型
- **建议**：先用免费版体验，觉得好用再升级

### Q：我的代码会被上传到云端吗？有隐私问题吗？

- Cursor 确实需要把你的代码片段发送到 AI 服务器来获取回复
- 如果你处理的是敏感代码（公司核心业务逻辑），建议咨询公司的信息安全部门
- Cursor 提供了 Privacy Mode（隐私模式），可以在设置中开启，它会限制发送的代码范围
- 对于产品经理自己练手搭原型、写文档来说，通常不涉及敏感信息，不需要太担心

### Q：AI 生成的代码能直接用到正式产品中吗？

- **原型验证**：可以放心用，原型的目的就是快速验证想法
- **正式产品**：不建议直接用。AI 生成的代码可能有安全漏洞或性能问题，需要研发 Review
- **文档**：AI 生成的文档建议自己审阅和修改后再发出去，确保准确性

### Q：我不小心搞坏了项目怎么办？

别慌，有几种方式恢复：
1. **Cmd+Z**：撤销最近的编辑操作
2. **Discard Changes**：在 Git 面板里丢弃某个文件的修改
3. **Git 回退**：回到之前的某次提交（在终端输入 `git log --oneline` 看历史，然后让 AI 帮你回退）
4. **最坏情况**：删除整个项目文件夹，从头再来（反正 AI 能帮你重新生成）

### Q：除了本课程教的，还有什么学习资源？

- **Cursor 官方文档**：搜索 "Cursor docs" 即可找到
- **YouTube / B 站**：搜索 "Cursor 教程"，有很多视频教程
- **多动手练**：最好的学习方式是给自己找一个小项目去做
