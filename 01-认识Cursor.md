# 第 1 课：认识 Cursor — 为什么产品经理需要它

## 本课目标

安装好 Cursor，认识它的界面，能打开一个文件夹开始工作。

---

## Cursor 是什么？

一句话版本：**Cursor 是一个内置了 AI 助手的代码编辑器。**

如果你用过 Notion，你应该见过 Notion AI 能帮你写文章、总结内容。Cursor 做的事情类似，只不过它的主场是「代码」——你用中文告诉它你想要什么，它帮你写代码、改代码、生成页面。

更准确地说，Cursor 是基于 VS Code（微软出的一款非常流行的代码编辑器）改造的，在上面加了一层强大的 AI 能力。所以它既能做所有 VS Code 能做的事情，又能让你用自然语言和 AI 对话来完成工作。

> 类比理解：VS Code 之于 Cursor，就像普通 Word 之于带 Copilot 的 Word。底子是一样的，但多了个超级聪明的 AI 助手。

---

## 产品经理用 Cursor 干什么？

你可能会想："我又不写代码，要这个干嘛？"

好问题。作为 PM，你用 Cursor 主要做两件事：

### 场景一：搭建可交互的产品原型

以前你做原型可能用 Figma 或者墨刀，画出来的是「图片」——看起来像真的，但点击跳转都是假的。用 Cursor，你可以让 AI 帮你生成真实的前端页面——有真实的按钮、能填写的输入框、能勾选的复选框。你甚至可以把这个页面发给同事，让他们在浏览器里直接操作。

这对验证想法、用户测试、向老板演示，都非常有帮助。

### 场景二：撰写产品方案文档

写 PRD、竞品分析、技术方案这些文档，你可以在 Cursor 里用 Markdown（一种简单的文本格式，后面会教）来写，AI 能帮你生成模板、填充内容、优化表达。比写在飞书或 Notion 里多了一个超级强的 AI 协作者。

---

## 下载和安装

1. 打开浏览器，访问 Cursor 官网（搜索 "Cursor editor" 即可找到）
2. 点击页面上的 **Download** 按钮，它会自动识别你的电脑系统（Mac 或 Windows）
3. 下载完成后：
   - **Mac 用户**：打开 .dmg 文件，把 Cursor 图标拖到「应用程序」文件夹
   - **Windows 用户**：双击 .exe 安装文件，一路点「下一步」
4. 首次打开 Cursor，它会让你注册账号或登录——用邮箱注册即可
5. 登录后，它可能会问你选择 AI 模型。选默认推荐的就好，后面可以随时改

> 小贴士：如果你之前用过 VS Code，Cursor 可以一键导入你的设置和插件。如果没用过，跳过这一步就行。

---

## 界面导览

打开 Cursor 之后，你会看到一个多区域的界面。别慌，我们一块块认识：

### 左侧：侧边栏（文件树）

这里显示的是你当前打开的文件夹里的所有文件和子文件夹，就像你电脑上的「访达」或「文件资源管理器」一样。你可以在这里点击文件来打开它。

### 中间：编辑区

这是你主要工作的地方。打开一个文件后，它的内容会显示在这里。你可以直接在这里修改文字、代码，或者让 AI 帮你改。

### 底部：终端（Terminal）

终端就是一个「命令行窗口」。你可以把它理解为**给电脑发送文字指令的地方**。比如你想启动一个网页项目来预览，就在这里输入命令。现在知道它在哪里就行，第 5 课会教你用。

### 右侧 / 弹出面板：AI Chat

按 `Cmd+L`（Mac）或 `Ctrl+L`（Windows）就能打开 AI 对话面板。这是你和 AI 交流的地方，可以问问题、让它写代码、让它改文件。

---

## 打开文件夹 = 打开项目

在 Cursor 里，「打开一个文件夹」就等于「打开一个项目」。

操作步骤：
1. 点击菜单栏的 **File**（文件）
2. 选择 **Open Folder**（打开文件夹）
3. 选择你电脑上的任意一个文件夹

选好之后，左侧的文件树就会显示这个文件夹里的所有内容。以后你要搭建原型或者写文档，第一步都是先创建一个空文件夹，然后在 Cursor 里打开它。

> 类比理解：在 Figma 里你先建一个「项目」，在 Notion 里你先建一个「页面」，在 Cursor 里你先打开一个「文件夹」。道理一样。

---

## 课后练习

1. 下载并安装 Cursor
2. 注册账号并登录
3. 在桌面上新建一个空文件夹，起个名字比如 `my-first-project`
4. 在 Cursor 里用 File → Open Folder 打开这个文件夹
5. 按 `Cmd+L` 打开 AI 面板，随便问它一个问题，比如"什么是 HTML？"——感受一下和 AI 对话的体验
